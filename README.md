# AutoArchiver

ファイル・フォルダを投げるだけで、**最適な圧縮形式の自動選択 → 最高圧縮 → 書庫テスト** まで一律でやってくれるWindows用ツール。

## 使い方

1. `AutoArchiver.exe` を起動してファイル・フォルダをドラッグ&ドロップ（複数可）、またはドロップゾーンをクリックしてフォルダ選択
2. 「送るメニューに登録」ボタンを押すと、右クリック →「送る」に2種類登録される
   - **AutoArchiver (自動圧縮)** — 選択したものをまとめて1書庫
   - **AutoArchiver (個別に自動圧縮)** — 1ファイル/フォルダごとに1書庫（バッチモード）
3. 圧縮＆書庫テスト成功後、元アイテムの扱いは設定に従う（毎回確認 / 常に残す / 常にゴミ箱）

出力書庫は元と同じ場所（または指定した保存先）に `名前.7z / .rar / .zip` として作られる（同名があれば連番）。
複数アイテムをまとめた場合の書庫名は共通の親フォルダ名。

## 機能

- **多重起動の合流** — 圧縮中に「送る」で追加しても、新しいウィンドウではなく既存のキューに追加される
- **完了通知** — 全キュー完了時に音+バルーン通知（設定でオフ可）
- **タスクバー進捗** — 最小化中もタスクバーアイコンに進捗表示
- **空き容量チェック** — 圧縮開始前に出力先ドライブの空きを予測サイズと比較、不足なら中止
- **除外パターン** — Thumbs.db / desktop.ini / .DS_Store を既定で書庫から除外（設定で編集可）
- **パスワード付き書庫** — 設定でパスワードを指定すると暗号化書庫に（7z/RARはファイル名も暗号化、DPAPI保存）
- **RARリカバリレコード** — 設定で0〜10%（既定0=なし）
- **保存先指定** — 未指定なら元と同じ場所、指定すれば固定の保存先へ

設定は `%APPDATA%\AutoArchiver\settings.json` に保存され、「送る」経由の起動にもそのまま適用される。

## 形式自動選択のロジック（ハイブリッド方式）

1. **拡張子ヒューリスティック（即決）** — 全ファイルをサイズ加重で分類
   - 既圧縮データ（動画・画像・音声・書庫等）が95%以上 → **無圧縮ZIP**（縮み代がないので時間を掛けない）
   - 圧縮可能データ（テキスト・ソース・実行ファイル等）が75%以上 → **7z LZMA2最高圧縮**（この構成はほぼ常勝）
2. **層化サンプルベンチ（混在・不明形式が多いとき）**
   - フォルダ全体から無作為サンプリングすると偏るため、**カテゴリごとに**サンプルを取って実測
   - カテゴリ別圧縮率をサイズ構成比で加重合成 → フォルダ全体の予測圧縮率を算出
   - 7z vs RAR5 の実測勝者を採用。ほぼ縮まない（98.5%以上）なら無圧縮ZIPへ
   - ゲームフォルダのような「画像・音声・テキスト・独自形式の混在」でも偏らない判定になる

## 圧縮設定

| 形式 | コマンド設定 | 用途 |
|------|-------------|------|
| 7z | `-mx=9 -ms=on`（LZMA2最高圧縮・ソリッド） | 圧縮可能データ主体・ベンチ勝者 |
| RAR5 | `-ma5 -m5 -md256m -s`（最高圧縮・辞書256MB・ソリッド） | ベンチ勝者のとき |
| ZIP | `-mx=0`（無圧縮 store） | 既圧縮データ主体のとき |

書庫テストは `7z t` / `Rar t` で実行し、**警告も不合格扱い**。不合格の書庫は削除して元フォルダを残す。

## 動作要件

- Windows 10 / 11
- .NET 8 Desktop Runtime
- [7-Zip](https://www.7-zip.org/)（必須: `C:\Program Files\7-Zip\7z.exe`）
- WinRAR（任意: あればRAR形式も候補に入る。無ければ7z/ZIPのみで動作）

## ビルド

```
dotnet build AutoArchiver.sln -c Release
```
